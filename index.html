投研 Pipeline

01
🚪 Deal Intake
投研入场路由
intake-card.md
+
接收任何形式的材料（BP、链接、描述），自动评估材料质量，判断当前阶段，推荐后续 skill 调用路径。

独到之处

判断当前所处阶段：初次接触 / 材料积累 / 准备 call / 深度尽调 / IC 准备
自动推荐后续应跑哪几个 skill、以什么顺序运行
在 deals/{公司名}/ 创建项目目录，所有分析产出自动归档

02
📄 Founder One-Pager
公司简介压缩
one-pager.md
+
将所有材料压缩成高密度一页简介：创始人背景、产品、traction、核心差异点、主要风险。
输入 → intake-card.md 原始材料
独到之处
强制区分【事实】【高概率推论】【观点】，不把创始人叙事写成事实
明确列出信息缺口：哪些关键维度还没有证据
涵盖：why now / moat / disconfirming evidence / 主要风险
如果 intake-card.md 已存在，自动读取作为背景，不需要重复粘贴材料
想获取 skill 请联系小林，并请他吃顿饭 🍜
目标是让任何人拿到这页纸，5 分钟内形成有质量的初步判断，而不是读完感觉"说了很多但没有结论"。
03
🗺️ Evidence Map
证据交叉验证
evidence-map.md
+
找出所有"说得通"和"说不通"的地方。哪些有多个来源印证？哪些只有创始人自己说？哪些互相矛盾？
输入 → one-pager.md 多份原始材料
独到之处
区分"多处提到"（可能只是同一个人重复说）与"有独立证据支持"
标出矛盾点：BP 说法 vs 访谈说法 vs 产品实测 vs 外部数据
明确列出：最值得优先验证的 3 个问题
防止被"材料多=信息多"的幻觉带偏
想获取 skill 请联系小林，并请他吃顿饭 🍜
材料越多，越容易产生虚假的置信度。Evidence Map 的作用不是总结材料，而是找出材料之间的裂缝。
04
⚔️ Critical Storyline Attack
反向压力测试
attack.md
+
如果这个项目会死，最可能死在哪？主动找最致命的攻击点，不被漂亮 narrative 带偏。
输入 → evidence-map.md one-pager.md
独到之处
找最致命的一个攻击点，而不是列举所有可能的风险
攻击角度：产品是否有独立存在价值？大模型公司下场会直接颠覆吗？Distribution 被高估了吗？
每个攻击点必须给出具体论据，不输出空洞的"竞争激烈"
默认站在反方立场，比任何 devil's advocate 都更彻底
想获取 skill 请联系小林，并请他吃顿饭 🍜
好的投资判断不只是找理由投，更要找到最硬的理由不投，然后决定是否能反驳它。
05
❓ Diligence Question List
尽调问题清单
question-list.md
+
根据所有已知信息和证据缺口，生成真正值得问的问题。按优先级排序，不问废话。
输入 → attack.md evidence-map.md
独到之处
自动读取所有前序文件，问题直接针对已发现的矛盾点和证据缺口
按优先级分级：P0 必问 / P1 重要 / P2 选问
每个问题注明：想验证什么 / 为什么这个问题信息增量高
不输出模板化客套问题，问题服务于判断质量提升
想获取 skill 请联系小林，并请他吃顿饭 🍜
与 founder 的每一轮 call 时间都是稀缺资源。用错问题是最大的浪费。这个 skill 确保每个问题都在增加信息量。
06
📋 Investment Memo
投资判断备忘录
investment-memo.md
+
综合 Pipeline 所有分析，给出结构化最终判断：thesis · risks · evidence gaps。
输入 → 全量分析文档 founder call 纪要
独到之处
自动读取目录下所有已有文件，综合作为 memo 基础，不让用户重复粘贴
必须包含反方观点（不能只有正面 thesis）
区分：已知事实 / 高概率推论 / 主观判断，结论的确定性层级清晰
给出明确结论：值不值得继续跟进，下一步行动是什么
想获取 skill 请联系小林，并请他吃顿饭 🍜
Memo 不是信息的堆砌，而是判断的文档化。它必须让读者在 10 分钟内知道：这个项目，投还是不投，为什么。

AGI Talent Intelligence
AI 人才情报
对 AI/AGI 领域从业者做背景调查、技术主张分析、职业轨迹追踪。生成结构化 profile。
人才 Profile · 背景调查
强制实体消歧：同名同姓的人在 AI 圈很多，先核实是哪个人，再生成 profile
技术主张分析：这个人真正相信什么？跟主流观点的差异在哪？
职业轨迹：从哪里来，往哪里走，转折点是什么
与 candidate-audit 的区别：本 skill 是外部情报（这人是谁），不是招聘判断（要不要招）

情报价值 ≠ 信息量。一份好的人才 profile 应该能告诉你：这个人的判断值不值得认真对待。

Market Sizing
市场规模估算
对市场规模、ARPU、留存、渗透率等做合理估算与 sanity check。优先给区间，明确口径。
TAM · ARPU · Sanity Check

Sanity check 创始人给的数字：10 亿用户 × 10 美元 ARPU 这种说法成立吗？
从已知参照物往上推：类比 Spotify、LinkedIn 等可验证的数据点

Section 03 · Documents

访谈与文档处理

从转写稿清洗到周报生成，完整覆盖文档生命周期。点击卡片查看详情。

🎙️
+
Transcript
逐字稿整理
将 ASR 自动转写稿（Recall、Otter、飞书妙记等）清洗为可读逐字稿。严格保持原意，不做摘要。
ASR 清洗 · 保持原意
清洗但不改写：去掉语气词、重复、明显的 ASR 错误，但不改变说话人的用词和逻辑
专名、术语、数字：仅在高置信度时纠正，否则保留并标注存疑
按话题分组：让长篇逐字稿有结构，不丢内容
完成后自动提示：要不要继续用 Meeting Consensus 提取共识和行动项
想获取 skill 请联系小林，并请他吃顿饭 🍜
逐字稿的价值在于保留说话人的原始语气和逻辑。过度整理会让你失去最重要的信号。
🤝
+
Meeting Consensus
会议共识提取
从会议纪要、聊天记录、项目复盘中提取共识、分歧与未决问题，以及明确的下一步行动项。
共识 · 分歧 · 行动项
区分：已形成共识 / 存在分歧 / 还没有讨论到的问题
提取好的体验和不好的体验（适用于产品复盘场景）
行动项必须带：负责人 / 截止时间 / 验收标准（能写多少写多少）
可直接衔接 Transcript skill：逐字稿清洗完，一句话触发出纪要
想获取 skill 请联系小林，并请他吃顿饭 🍜
好的纪要不是把会议内容再说一遍，而是让没参会的人5 分钟内知道结论、分歧和下一步。
